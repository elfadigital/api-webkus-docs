Dokumentasi API Webkus (Unofficial)

by Elfa Digital

Traktir Admin Kopi

Dokumentasi ini dibuat berdasarkan hasil rekayasa balik (reverse-engineering) dari situs web elfa-dev.webkus.com (Webkus v3.4) melalui serangkaian panggilan API nyata pada tanggal 11 Maret 2026. Gunakan sebagai panduan untuk memahami bagaimana cara berinteraksi dengan backend Webkus.

Base URL

https://{nama-toko-anda}.webkus.com

Konsep Kunci: local_id

Sebagian besar interaksi API yang terkait dengan pengguna (seperti keranjang belanja) tidak menggunakan autentikasi akun pengguna tradisional. Sebagai gantinya, Webkus mengidentifikasi setiap pengunjung unik menggunakan sebuah local_id.

Apa itu local_id?

Sebuah string unik yang secara otomatis dibuat oleh Webkus untuk setiap pengunjung baru.

Penyimpanan

Disimpan di dalam localStorage browser pengunjung. Digunakan untuk mengambil data keranjang kembali.

Cara Mendapatkan

Panggil endpoint GET /api/get-data/-. Tanda - menandakan permintaan ID baru.

1. Data Toko & Produk

Endpoint ini digunakan untuk mengambil informasi publik mengenai toko dan produk.

GET

/api/info

Informasi Konfigurasi Dasar

Mengambil informasi konfigurasi dasar sebuah toko (nama toko, deskripsi, kontak, logo).

Contoh Request (cURL) 

curl -L "https://elfa-dev.webkus.com/api/info"

Respons Sukses (200 OK)

GET

/api/get-data/{local_id}

Muat Data & Mulai Sesi

Endpoint utama untuk memuat data halaman depan dan memulai sesi pengguna.

Parameter URL

  • local_id
    (string, wajib) ID unik pengunjung. Gunakan - untuk mendapatkan ID baru.

Alur Implementasi

  1. Saat halaman dimuat, periksa localStorage untuk local_id.
  2. Jika tidak ada, panggil menggunakan - (GET /api/get-data/-).
  3. Jika ada, panggil menggunakan ID tersimpan (GET /api/get-data/ID_YANG_TERSIMPAN).
  4. Simpan local_id dari respons ke storage untuk panggilan selanjutnya.

Contoh Request (ID Baru) 

curl -L "https://elfa-dev.webkus.com/api/get-data/-"

2. Manajemen Keranjang (Cart)

Semua endpoint di bawah ini memerlukan local_id untuk mengidentifikasi keranjang milik siapa.

POST

/api/carts/{local_id}

Tambah Item ke Keranjang

Menambahkan sebuah produk ke dalam keranjang. Jika product_id tersebut sudah ada di dalam keranjang, maka panggilan ini akan menambah jumlah (quantity) produk tersebut.

Headers

Content-Type: application/json

Payload

{
  "product_id": "ID_PRODUK_YANG_AKAN_DITAMBAH"
}

cURL Simulasi 

curl -L -X POST "https://elfa-dev.webkus.com/api/carts/1773180573477_829959" \
-H "Content-Type: application/json" \
-d '{"product_id": "e7i9ybiq"}'

Respons Sukses

{
  "cart_count": 1
}

*Catatan: cart_count adalah jumlah jenis/item unik, bukan total kuantitas.

GET

/api/carts/{local_id}

Lihat Detail Keranjang

Mengambil daftar item yang ada di dalam keranjang seorang pengunjung. id di level atas pada setiap item adalah cart_item_id yang digunakan untuk operasi PATCH/DELETE.

PATCH Ubah Catatan Item

/api/carts/{local_id}/{cart_item_id}

Mengubah catatan (message) pada sebuah item di keranjang.

Payload

{
  "message": "Ini adalah catatan baru untuk item ini."
}

Contoh Request

curl -L -X PATCH "https://elfa-dev.webkus.com/api/carts/1773180573477_829959/4qiaxe7iq" \
-H "Content-Type: application/json" \
-d '{"message": "Catatan baru."}'
DELETE Kurangi Qty / Hapus

/api/carts/{local_id}/{cart_item_id}

Mengurangi jumlah (quantity) item. Jika quantity mencapai 1, maka panggilan ini akan menghapus item dari keranjang.

Contoh Request

curl -L -X DELETE "https://elfa-dev.webkus.com/api/carts/1773180573477_829959/4qiaxe7iq"

Respons (Raw)

{
  "cart": {
    "id": "4qiaxe7iq",
    "local_id": "1773180573477_829959",
    "product_id": "e7i9ybiq",
    "quantity": 1,
    "message": "",
    "created_at": "2026-03-10T22:10:18.561Z"
  }
}

3. Pesanan & Checkout

Proses terakhir dalam alur transaksi dari keranjang menjadi pesanan nyata.

POST

/api/checkout/{local_id}

Buat Pesanan Baru

Membuat pesanan baru dari semua item yang ada di keranjang seorang pengunjung.

Payload Data Pembeli

{
  "name": "Nama Pelanggan",
  "email": "email@pelanggan.com",
  "phone": "081234567890"

GET

/api/orders/{order_id}

Detail & Status Pesanan

Mengambil detail sebuah pesanan, baik yang masih pending maupun yang sudah completed. Perbedaan utamanya ada pada data pengiriman digital yang akan muncul jika pesanan sudah selesai dibayar.

Contoh: Pending Menunggu Pembayaran

  "order": {
    "id": "4qiax4ciqnuppel",
    "status": "pending",
    "name": "Budi Tester",
    "email": "budi.tester@example.com",
    "phone": "081234567891",
